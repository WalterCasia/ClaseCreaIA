### Agregando habilidades

```python
class Vehiculo:
    def __init__(self, marca, modelo):
        self.marca = marca
        self.modelo = modelo

    def arrancar(self):
        print(f"El {self.marca} {self.modelo} ha arrancado.")

class Coche(Vehiculo):
    def __init__(self, marca, modelo, num_puertas):
        super().__init__(marca, modelo)
        self.num_puertas = num_puertas

    def abrir_maletero(self):
        print(f"El maletero del {self.marca} {self.modelo} se ha abierto.")

# Uso
coche = Coche("Toyota", "Corolla", 4)
coche.arrancar()  # Método heredado
coche.abrir_maletero()  # Método propio
```

## metodo Super()

super(). significa literalmente superclase

```python
class Persona:
    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad = edad

    def saludar(self):
        print(f"Hola, mi nombre es {self.nombre} y tengo {self.edad} años.")

class Estudiante(Persona):
    def __init__(self, nombre, edad, carrera):
        super().__init__(nombre, edad)
        self.carrera = carrera

    def estudiar(self):
        print(f"Estoy estudiando {self.carrera}.")

# Uso
estudiante = Estudiante("Juan", 20, "Ingeniería")
estudiante.saludar()  # Método heredado
estudiante.estudiar()  # Método propio
```

# Las muchas formas (Polimorfismo)

**SOLID** -> Principio de sustitución de Liskov
**S - Single Responsibility Principle** -> Principio de responsabilidad única (Una clase debe tener una sola razón para cambiar)
**O - Open/Closed Principle** -> Principio de abierto/cerrado (Una clase debe estar abierta para extensión, pero cerrada para modificación)
**L - Liskov Substitution Principle** -> Principio de sustitución de Liskov (Una clase debe poder ser reemplazada por su subclase sin romper el programa)
**I - Interface Segregation Principle** -> Principio de segregación de interfaces (Una clase no debe ser forzada a implementar métodos que no usa)
**D - Dependency Inversion Principle** -> Principio de inversión de dependencias (Una clase debe depender de abstracciones, no de implementaciones concretas)
