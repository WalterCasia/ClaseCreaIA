## 1. El Constructor (`__init__`)

El constructor es un método especial que **se ejecuta automáticamente en el instante exacto en que creamos (instanciamos) el objeto**. 

**La Analogía de la Receta:**
Imagina que el plano (la Clase) es una receta para hornear un pastel. El constructor (`__init__`) es el paso donde mezclas los ingredientes. No puedes hornear un pastel de chocolate y luego, media hora después de sacarlo del horno, intentar meterle el chocolate. Los ingredientes deben ingresar al mismo tiempo que se crea el pastel.

### Sintaxis
`def __init__(self):`

*Nota mental:* La palabra `self` significa "yo mismo" o "mi propio". Cuando escribimos `self.titulo`, la computadora entiende "mi propio título".

### Ejemplo: La Fábrica de Libros


```python
class Libro:
    # Nuestro constructor exige dos ingredientes externos (parámetros) obligatorios
    def __init__(self, titulo_ingresado, autor_ingresado):
        # Guardamos los ingredientes que vienen de afuera en el interior del objeto (atributos)
        self.titulo = titulo_ingresado
        self.autor = autor_ingresado

# LA FABRICACIÓN (Instanciación)
# Aquí vamos a enviar los ingredientes a nuestra clase para crear un objeto
# Estos textos viajan directamente a los paréntesis del __init__
libro_nuevo = Libro("El Principito", "Antoine de Saint-Exupéry")
libro_viejo = Libro("Don Quijote", "Miguel de Cervantes")

# COMPROBACIÓN
# Cada libro recuerda sus propios datos gracias a la palabra 'self'
print(f"El autor registrado del libro '{libro_nuevo.titulo}' es {libro_nuevo.autor}")
print(f"El autor registrado del libro '{libro_viejo.titulo}' es {libro_viejo.autor}")
```

## 2. Métodos con Parámetros Normales

No todos los datos deben ingresarse al nacer. Algunos objetos nacen "vacíos" y van cambiando a lo largo de su vida usando métodos (acciones).

```python
class Alcancia:
    # Al nacer, esta alcancía no pide ingredientes de afuera
    def __init__(self):
        # Todos los objetos de esta clase nacen con 0 en su interior
        self.dinero_guardado = 0

    # Este método sí necesita un parámetro externo para saber cuánto dinero sumar
    def depositar_dinero(self, cantidad):
        # Sumamos el parámetro externo al atributo interno
        # self.dinero_guardado += cantidad hace exactamente lo mismo que la línea de abajo
        self.dinero_guardado = self.dinero_guardado + cantidad

    def mostrar_ahorros(self):
        print(f"El dinero total ahorrado es: {self.dinero_guardado}")


# 1. Crear el objeto (nace con 0)
mi_chanchito = Alcancia()

# 2. Acciones a lo largo del tiempo
mi_chanchito.depositar_dinero(500)
mi_chanchito.depositar_dinero(1200)

# 3. Comprobación
mi_chanchito.mostrar_ahorros() # Resultado esperado: 1700
```

## 3. Ejercicio Práctico: El Cajero Automático

**Instrucciones:**
Crea la clase `Cajero` siguiendo estas reglas:
1. Su constructor debe pedir de afuera el `nombre` y el `id_empleado`.
2. Debe tener un atributo interno llamado `dinero_en_caja` que inicie siempre en 0.
3. Crea un método `cobrar_articulo(self, precio_articulo)` que le sume el precio recibido al dinero de la caja.
4. Crea un método `mostrar_dinero_caja(self)` que imprima el dinero total junto con el nombre y el id del cajero.
5. Crea dos cajeros con datos inventados.
6. Cóbrale 2 artículos al cajero 1 y 1 artículo al cajero 2.
7. Muestra el dinero final de ambas cajas.

### Solución del Ejercicio:
```python
class Cajero:
    def __init__(self, nombre, id_empleado):
        self.nombre = nombre
        self.id_empleado = id_empleado
        self.dinero_en_caja = 0
        
    def cobrar_articulo(self, precio_articulo):
        self.dinero_en_caja += precio_articulo
        print(f"Artículo de {precio_articulo} cobrado por {self.nombre}.")
        
    def mostrar_dinero_caja(self):
        print(f"Cajero: {self.nombre} (ID: {self.id_empleado}) | Total en caja: {self.dinero_en_caja}")

# Creación de objetos
cajero1 = Cajero("Ana", "EMP-001")
cajero2 = Cajero("Luis", "EMP-002")

# Operaciones
print("--- Turno de la Mañana ---")
cajero1.cobrar_articulo(1500)
cajero1.cobrar_articulo(3000)

cajero2.cobrar_articulo(5000)

print("\n--- Cierre de Cajas ---")
cajero1.mostrar_dinero_caja()
cajero2.mostrar_dinero_caja()
```

## 4. Parámetros Opcionales en el Constructor

A veces queremos que un parámetro tenga un valor "por defecto" por si el usuario olvida enviarlo o si la mayoría de las veces es el mismo. Si el usuario envía un dato, el valor por defecto se borra y se usa el nuevo.

```python
class Cliente:
    # El país por defecto será Guatemala, a menos que nos digan lo contrario en los paréntesis
    def __init__(self, nombre, pais="Guatemala"):
        self.nombre = nombre 
        self.pais = pais

# Le enviamos ambos datos. Se ignora "Guatemala" y se usa "Costa Rica".
cliente1 = Cliente("Diego", "Costa Rica")
print(cliente1.pais) # Imprime: Costa Rica

# Solo enviamos el nombre. Como falta el segundo dato, Python usa "Guatemala".
cliente2 = Cliente("Kevin")
print(cliente2.pais) # Imprime: Guatemala
```

## 5. Modificando Atributos a través de Métodos (Buenas Prácticas)

**La Analogía del Control Remoto:**
Un objeto debe proteger su información interna. Si quieres cambiar el canal de un televisor, no sacas un destornillador, abres la pantalla y mueves los cables adentro. Eso es peligroso (Mala práctica). Lo correcto es usar los botones del control remoto (los Métodos). Los métodos se aseguran de que las reglas lógicas se cumplan (por ejemplo, no puedes subir el volumen si el televisor está apagado).


```python
class Televisor: 
    def __init__(self, marca):
        self.marca = marca
        self.encendido = False # Estado inicial: apagado
        self.volumen = 10      # Estado inicial: volumen 10
    
    def presionar_boton_encendido(self):
        # Si estaba apagado (False), lo prendemos (True)
        # if not self.encendido: es exactamente lo mismo que if self.encendido == False:
        if not self.encendido:     
            self.encendido = True
            print(f"El televisor {self.marca} se ha encendido.")
        # Si estaba encendido (True), lo apagamos (False)
        else:
            self.encendido = False
            print(f"El televisor {self.marca} se ha apagado.")

    def subir_volumen(self):
        # Primero validamos si está encendido antes de permitir subir el volumen
        if self.encendido:         
            self.volumen += 1
            print(f"El volumen actual es: {self.volumen}")
        else:
            print("Error: No se puede subir el volumen. El televisor está apagado.")
        
tv1 = Televisor("Samsung")

# tv1.volumen = 50  -> ¡ESTO ES UNA MALA PRÁCTICA! No abras el televisor con destornillador.
# Lo correcto es usar el método (el control remoto):
tv1.subir_volumen() # Dará error porque está apagado

# tv1.encendido = True -> ¡ESTO TAMBIÉN ES UNA MALA PRÁCTICA!
# Lo correcto es usar su botón:
tv1.presionar_boton_encendido() # Ahora sí está encendido

# Ahora los botones funcionarán correctamente
tv1.subir_volumen()
tv1.subir_volumen()

# Apagamos el televisor
tv1.presionar_boton_encendido()
```

## 6. Reto Único Integrador: El Personaje de Videojuego


* **Instrucción:** Vas a programar la lógica del jugador para un videojuego de rol (RPG).
    1. Crea la clase `Personaje`.
    2. Crea el método `__init__`. Este debe pedir como parámetro externo el `nombre` del personaje.
    3. Dentro del `__init__`, asígnale los siguientes atributos por defecto (sin pedirlos en los paréntesis):
        * `self.nivel = 1`
        * `self.puntos_de_vida = 100`
        * `self.esta_vivo = True`
    4. Crea un método llamado `recibir_dano(self, cantidad_dano)`.
        * Este método debe restarle la `cantidad_dano` a los `puntos_de_vida`.
        * Si los puntos de vida llegan a 0 o menos, el estado `esta_vivo` debe cambiar a `False`, y debe imprimir: "¡[nombre] ha sido derrotado!".
        * Si aún le queda vida, debe imprimir: "[nombre] recibió daño. Vida restante: [puntos_de_vida]".
    5. Fuera de la clase, instancia a tu personaje dándole un nombre. 
    6. Atácalo dos veces usando el método `recibir_dano` (ej: pásale 40 de daño, y luego 70 de daño) para comprobar que su estado cambia de vivo a muerto.

### Solución del Reto:
```python
class Personaje:
    def __init__(self, nombre):
        self.nombre = nombre
        # Atributos internos por defecto (no se piden al usuario)
        self.nivel = 1
        self.puntos_de_vida = 100
        self.esta_vivo = True

    def recibir_dano(self, cantidad_dano):
        # Validamos primero si el personaje ya está derrotado
        if not self.esta_vivo:
            print(f"{self.nombre} ya está derrotado, no puedes hacerle más daño.")
            return # Termina la ejecución del método aquí

        # Restamos la vida
        self.puntos_de_vida -= cantidad_dano
        
        # Evaluamos si sobrevivió al ataque
        if self.puntos_de_vida <= 0:
            self.puntos_de_vida = 0 # Evitamos vidas negativas
            self.esta_vivo = False
            print(f"¡{self.nombre} ha recibido un golpe letal y ha sido derrotado!")
        else:
            print(f"{self.nombre} recibió {cantidad_dano} de daño. Vida restante: {self.puntos_de_vida}")

# 1. Creación del personaje
heroe = Personaje("Arthur")
print(f"El héroe {heroe.nombre} de nivel {heroe.nivel} ha entrado a la arena con {heroe.puntos_de_vida} HP.")

# 2. Simulador de combate
print("\n--- ¡Inicia el combate! ---")
heroe.recibir_dano(40)
heroe.recibir_dano(70)

# 3. Comprobación final
print("\nEstado final del héroe:")
print(f"¿Está vivo?: {heroe.esta_vivo}")
print(f"Vida actual: {heroe.puntos_de_vida}")
```
